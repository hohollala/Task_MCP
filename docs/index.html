MCP Task Manager

체계적인 프로젝트 관리를 위한 MCP 서버

📋 개요

MCP Task Manager는 AI 개발자와 함께 체계적으로 프로젝트를 관리할 수 있도록 도와주는 Model Context Protocol (MCP) 서버입니다. 복잡한 개발 프로젝트를 단계별로 분해하고, 진행 상황을 추적하며, 효율적인 작업 관리를 가능하게 합니다.

✨ 주요 기능

체계적인 요구사항 분석

7가지 핵심 질문을 통해 프로젝트 요구사항을 체계적으로 수집하고 분석합니다.

단계별 작업 계획

복잡한 프로젝트를 대분류-중분류-소분류 구조로 체계적으로 분해합니다.

진행 상황 추적

작업 상태를 실시간으로 추적하고 다음 작업을 자동으로 안내합니다.

자동 디자인 문서화

UI/UX 작업 시 자동으로 디자인 가이드와 문서를 생성합니다.

5단계 사고 프로세스

요구사항 분석부터 구현까지 AI 개발자 사고방식을 적용합니다.

자동화된 워크플로우

작업 완료 감지 및 다음 작업 자동 진행으로 효율성을 극대화합니다.

🚀 시작하기

1. 설치

# 프로젝트 클론 git clone https://github.com/hohollala/Task_MCP.git cd Task_MCP # 한 번에 설치 및 설정 npm install && npm run build && npm install -g . # MCP 서버 등록 (Windows) claude mcp add task-manager -s user -- node $PWD/dist/index.js

2. 명령어 사용법

MCP Task Manager는 4가지 주요 명령어를 제공합니다:

📱 task-new

새 프로젝트의 요구사항을 체계적으로 수집합니다.

/task-new

생성 파일: docs/requirements.md, docs/designed.md, docs/technical_spec.md

📋 task-plan

요구사항을 기반으로 체계적인 프로젝트 계획을 수립합니다.

/task-plan

생성 파일: docs/project_task.md

🚀 task-start

프로젝트 계획에 따라 다음 작업을 시작합니다.

/task-start

기능: 작업 상태 추적, 자동 완료 감지, 다음 작업 진행

📋 task-resume

이전 작업을 이어서 진행합니다.

/task-resume

기능: 진행 상황 분석, 다음 미완료 작업 자동 시작

📁 파일 구조

MCP Task Manager는 다음과 같은 파일 구조를 생성하고 관리합니다:

docs/ ├── requirements.md # 프로젝트 요구사항 ├── designed.md # 디자인 가이드 ├── technical_spec.md # 기술 사양서 ├── project_task.md # 프로젝트 작업 계획 └── design.md # UI/UX 디자인 문서 (자동 생성)

작업 상태 표시

  • [ ] 대기중: 아직 시작하지 않은 작업
  • [-] 진행중: 현재 작업 중인 작업
  • [x] 완료: 완료된 작업

💡 사용 예시

전체 워크플로우

1단계: 요구사항 정의

/task-new

7가지 질문에 답변하여 프로젝트 요구사항을 정의합니다.

2단계: 계획 수립

/task-plan

요구사항을 분석하여 체계적인 프로젝트 계획을 수립합니다.

3단계: 작업 진행

/task-start

계획에 따라 작업을 시작하고 진행 상황을 추적합니다.

4단계: 작업 재개

/task-resume

이전 작업을 이어서 진행하거나 다음 작업을 시작합니다.

💡 팁: 각 단계에서 생성되는 문서들을 참고하여 프로젝트를 체계적으로 관리하세요.

🔧 기술 스펙

  • 언어: TypeScript
  • 프로토콜: Model Context Protocol (MCP)
  • 런타임: Node.js 16.0.0 이상
  • 의존성: @modelcontextprotocol/sdk, zod
  • 라이선스: MIT

🎯 주요 특징

체계적인 작업 분해

복잡한 프로젝트를 대분류 → 중분류 → 소분류 구조로 체계적으로 분해하여 관리하기 쉽게 만듭니다.

자동화된 진행 관리

작업 완료를 자동으로 감지하고 다음 작업을 자동으로 진행하여 효율성을 극대화합니다.

스마트한 사용자 확인

중요한 분기점에서만 사용자 확인을 요청하고, 세부 작업은 자동으로 연속 진행합니다.

통합 문서화

요구사항, 디자인, 기술 사양을 자동으로 문서화하여 프로젝트의 완성도를 높입니다.

⚠️ 주의사항

중요:
  • 프로젝트 시작 전 반드시 /task-new로 요구사항을 정의하세요.
  • 작업 완료 시 정확한 완료 상태를 확인하세요.
  • 생성된 문서들을 정기적으로 검토하고 업데이트하세요.
  • 복잡한 프로젝트의 경우 단계별로 나누어 진행하세요.
